md_storage_html — User Guide

md_storage_html is a desktop Java application for managing Markdown files and exporting them as a linked HTML website. It provides a two-pane editor, a file/folder tree, and one-click HTML export with an auto-generated index page.

Requirements

  • Java 11 or later
  • All .java source files in the same directory

Compiling & Running

cd /path/to/proj_03_md_storage
javac *.java
java md_storage_html

Application Layout

The window is divided into three areas:

  • Toolbar — action buttons and folder path labels across the top.
  • Left pane — file/folder tree (the Index).
  • Right pane — Markdown text editor for the selected file.

Toolbar Buttons

ButtonWhat it does
FilePlaceholder for future file menu actions.
SaveWrites all files and folders from the tree to the MD folder on disk, preserving the tree hierarchy.
md folderOpens a folder picker to set the MD source folder — where Markdown files are stored. If the chosen folder already contains .md files, they are loaded into the tree automatically.
html folderOpens a folder picker to set the HTML export folder — where generated HTML files will be written.
ExportConverts every .md file in the MD folder to HTML and writes it to the HTML folder, mirroring the folder structure. Also generates index.html at the root.
Open In BrowserOpens index.html from the HTML folder in the system default browser.
AboutShows the author contact information.

After selecting either folder, its full path is displayed next to a vertical separator on the right side of the toolbar for quick reference. Paths are also saved automatically to md_storage_html.properties and restored the next time the application starts.

Working with the File Tree (Left Pane)

Index root

The root node is labelled Index. All files and folders live under it.

Adding a file

Click + File below the tree, or right-click any node and choose Add File. Enter a file name — the .md extension is added automatically if omitted. The new file is inserted under the currently selected folder (or the root if nothing is selected).

Adding a folder

Click + Folder or right-click → Add Folder. Folders can be nested to any depth. Newly created folders display a system folder icon.

Renaming

Select a node, then click Rename or right-click → Rename. The current name is pre-filled. For files, the .md extension is preserved automatically.

Deleting

Select a node and click - Delete or right-click → Delete. A confirmation dialog is shown. Deleting a folder removes all its children from the tree and from memory.

Selecting a file

Click any file node to load its content into the right pane editor.

Editing (Right Pane)

The right pane is a plain text editor. Write standard Markdown syntax here. Content is kept in memory until you click Save, which flushes everything to disk.

Tip: The currently open file's content is automatically flushed to memory before Save writes to disk, so you do not need to click away first.

Loading an Existing Folder

Click md folder and select a directory that already contains .md files and sub-folders. The tree is rebuilt from disk automatically, preserving the original hierarchy. Only .md files are loaded; other file types are ignored.

Exporting to HTML

  1. Set the md folder (source).
  2. Set the html folder (destination).
  3. Click Export.

The exporter:

  • Mirrors the source folder tree inside the HTML folder.
  • Converts each .md file to a fully self-contained .html page.
  • Generates index.html at the root of the HTML folder listing every page with clickable links and an Open in new tab button.

HTML font size

The body font size of exported HTML pages is controlled by the html.font.size property in md_storage_html.properties. The default is 20 (px). Edit the value and re-run Export to apply it. 

html.font.size=20

The exported HTML is mobile-responsive — it uses a viewport meta tag, fluid layout, and a max-width: 600px media query to adjust font sizes on small screens.

Configuration File

Settings are stored in md_storage_html.properties in the same directory as the application. The file is created automatically on first use.

KeyDescription
md.folderAbsolute path to the Markdown source folder.
html.folderAbsolute path to the HTML export folder.
html.font.sizeBase font size (px) for exported HTML pages. Default: 20.
Note: The properties file is read at startup and written automatically whenever you change either folder. You can also edit it manually while the application is closed.
